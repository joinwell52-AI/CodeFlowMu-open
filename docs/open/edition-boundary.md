# 版本边界

公开版不是母体仓库的完整镜像，而是由母体仓库按版本清单生成的开源发行包。

核心定位：

- 母体仓库 `codeflowmu` 是 CodeFlowMu 自身的开发体；
- 公开仓库 `CodeFlowMu-open` 是 FCoP 协议下的团队开发应用工具；
- 公开版用于开发 Panel 当前业务项目；默认 `projects/newproject` 可直接使用，也可新建独立项目或添加已有源码。旧版 `workspace/<项目>` 保持原地兼容。
- `CodeFlowMu-open` 应用代码由自保护壳维护启动基线，业务项目目录不受该保护限制。

公开版包含：

- 面向本地体验的 PC Panel 和基础静态资源
- 固定开发团队：PM / DEV / OPS / QA；EVAL 为独立观察者
- Git 状态、项目配置、任务、报告、审批、聊天等基础协作流程
- 安装说明、版本边界、网关策略和贡献引导
- 工具根保护：公开版不能把自身目录注册为开发项目
- 项目隔离：每个项目使用自己的 `.codeflowmu/runtime`，切换项目会同步重绑 Runtime、MCP、Watcher 和 ledger
- 关单观察：PM 最终 REPORT 自动生成 EVAL，人工按钮只负责重试或刷新
- Gateway 边界：允许只读比较官方演示 PWA 版本，禁止从 Open 发布或覆盖线上 Gateway 静态资源

公开版不包含：

- 私有网关凭据
- 母体真实运行历史
- 私有日志、真实报告、真实任务账本
- 内部评估、内部发布、公司私有运营功能
- 尚未开放的数字员工实验功能

母体仓库负责版本选择、构建、同步、清理运行态、提交公开仓库和推送公开仓库。公开仓库负责下载、安装、介绍、体验和贡献入口。
