# UI Requirements

Use this playbook when UI work needs to turn rough feedback into clear interface requirements.

## Output

```text
# UI Requirements

## 1. User / Role
ADMIN / PM / DEV / QA / OPS / EVAL.

## 2. Workflow
What the user is trying to complete.

## 3. Current Problem
What is confusing, slow, hidden, misleading, or broken.

## 4. Desired Behavior
What should be visible, selectable, actionable, or prevented.

## 5. State Semantics
What each visible state means in FCoP terms.

## 6. Constraints
Panel density, mobile, accessibility, project root, lifecycle, privacy.

## 7. Acceptance Criteria
Checkable UI behavior.
```

## Rules

- Tie UI copy to real FCoP semantics.
- Do not describe features that are not implemented as if they are.
- Keep ADMIN control actions explicit.
