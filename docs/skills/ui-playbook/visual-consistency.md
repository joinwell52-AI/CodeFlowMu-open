# UI Visual Consistency

Use this playbook when reviewing or designing CodeFlowMu UI visuals and components.

## Output

```text
# Visual Consistency Review

## 1. Surface
Screen, panel, component, or modal.

## 2. Component Pattern
Existing pattern to match.

## 3. Layout
Spacing, density, grouping, responsive behavior.

## 4. Controls
Buttons, icons, tabs, filters, tables, cards, inputs.

## 5. Typography and Copy
Readable, non-overlapping, truthful.

## 6. Findings

| Issue | Impact | Suggested fix |
|---|---|---|
```

## Rules

- Prefer familiar controls and icons for common actions.
- Do not create card-in-card layouts.
- Avoid text overlap and viewport-scaled type.
- Keep operational UI calm and scannable.
