# QA Reproduce Issue

Use this playbook when QA must reproduce a bug or verify that a reported issue is real.

## Output

```text
# Reproduce Issue

## 1. Reported Behavior
What was reported.

## 2. Environment
Browser, OS, command, branch, data, or project root.

## 3. Steps
Numbered reproduction steps.

## 4. Actual Result
Observed behavior with evidence.

## 5. Expected Result
Expected behavior.

## 6. Reproducibility
Always / intermittent / not reproduced.
```

## Rules

- Reproduce before diagnosing when feasible.
- Keep evidence concise.
- Do not mark fixed from code inspection alone.
