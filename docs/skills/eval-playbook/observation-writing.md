# EVAL Observation Writing

Use this playbook when EVAL records an observation about behavior, UX, governance, or implementation gaps.

## Output

```text
# Observation

## 1. Observation
What was seen.

## 2. Evidence
Files, UI state, logs, screenshots, or commands.

## 3. Impact
Why it matters.

## 4. Confidence
High / medium / low.

## 5. Suggested Follow-up
Local task / PM review / OPS check / QA verification / hold.
```

## Rules

- Observations are not REPORTs.
- Observations do not change lifecycle.
- Do not overclaim without evidence.
