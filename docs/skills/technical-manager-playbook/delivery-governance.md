# Technical Manager Delivery Governance

Use this playbook when a technical manager needs to coordinate delivery across DEV, QA, OPS, PM, and EVAL without becoming ADMIN.

## Output

```text
# Technical Delivery Governance

## 1. Delivery Objective
What technical outcome is being coordinated.

## 2. Workstreams

| Workstream | Owner | File scope | Dependency | Acceptance |
|---|---|---|---|---|

## 3. Execution Order
Which work must happen first and why.

## 4. Handoff Points
Where TASK, REPORT, QA, PM review, or ADMIN decision is needed.

## 5. Risk Register
Technical risks and mitigation.

## 6. Escalation
What requires PM or ADMIN decision.
```

## FCoP Boundaries

- Technical manager coordinates; ADMIN accepts governance decisions.
- Do not archive or move lifecycle from a playbook.
- Do not assign unrelated file scope.
- Do not turn coordination into autonomous governance.
