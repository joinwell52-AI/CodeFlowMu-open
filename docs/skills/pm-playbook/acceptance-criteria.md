# PM Acceptance Criteria

Use this playbook to turn "done" into checkable acceptance conditions before dispatch, PM review, QA verification, or ADMIN closure.

## Output

```text
# Acceptance Criteria

## 1. Functional Acceptance

- [ ] Condition 1
- [ ] Condition 2

## 2. File Acceptance

- [ ] Only allowed files changed
- [ ] Forbidden files untouched

## 3. Behavior Acceptance

- [ ] UI copy matches true semantics
- [ ] Lifecycle was not bypassed
- [ ] REPORT / EVAL / TASK were not mixed

## 4. Evidence Acceptance

- [ ] REPORT exists
- [ ] Test results exist
- [ ] Logs or screenshots exist where applicable
- [ ] Blocked information exists where applicable

## 5. Forbidden Item Acceptance

- [ ] No adopted-pending 0003
- [ ] No automatic GitHub submission
- [ ] No automatic archive
- [ ] No automatic deletion
```
