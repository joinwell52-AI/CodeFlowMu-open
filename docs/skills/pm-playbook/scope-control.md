# PM Scope Control

Use this playbook to keep work bounded and prevent agents from expanding a task because adjacent improvements look tempting.

## Output

```text
# Scope Control

## 1. Current Task Goal
One sentence.

## 2. Allowed Change Scope
Files, modules, docs, and behavior that may change.

## 3. Forbidden Change Scope
Files, modules, docs, and behavior that must not be touched.

## 4. Non-goals
Problems this round will not solve.

## 5. Risk Expansion Points
Where scope can easily grow.

## 6. Suggested Split
Work that should become a separate task.
```
