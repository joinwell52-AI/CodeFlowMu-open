# External Skill Sources

This document records external GitHub sources that can inspire CodeFlowMu Playbook skills.

The policy is strict: use external repositories for ideas, structures, checklists, and vocabulary only. Do not copy whole repositories into `skills/`, do not enable external scripts, and do not treat external skills as CodeFlowMu runtime behavior.

## Sources

| Source | Use in CodeFlowMu |
|---|---|
| `https://github.com/deanpeters/Product-Manager-Skills` | Product requirement, roadmap, prioritization, and acceptance structure inspiration. |
| `https://github.com/product-on-purpose/pm-skills` | PM methodology inspiration. |
| `https://github.com/addyosmani/agent-skills` | Agent skill packaging and architecture review inspiration. |
| `https://github.com/github/awesome-copilot` | Broad Copilot/agent skill discovery. |
| `https://github.com/VoltAgent/awesome-agent-skills` | Broad agent-skill discovery. |

## Absorption Rules

- Rewrite every adopted idea for CodeFlowMu and FCoP boundaries.
- Keep external references in docs or manifest metadata, not as executable runtime code.
- Do not import external scripts.
- Do not add Panel/API behavior from external repositories.
- Do not change existing PM runtime skill IDs.
- Do not let external playbooks bypass ADMIN, lifecycle authority, or public-submission safety.
