# PM Delivery Plan

Use this playbook to split a requirement into executable, reviewable, and deliverable P0 / P1 / P2 work.

## Output

```text
# Delivery Plan

## 1. Goal
What this delivery solves.

## 2. Stages

| Stage | Task | Owner | File scope | Acceptance criteria |
|---|---|---|---|---|

## 3. P0
Must happen first.

## 4. P1
Do next.

## 5. P2
Do later.

## 6. Forbidden Items
What must not happen.

## 7. Risks and Dependencies
List blockers and dependencies.
```

## FCoP Boundaries

- PM may split tasks.
- PM does not replace final ADMIN confirmation.
- PM does not submit public issues automatically.
- PM must not assign downstream agents unrelated files.
