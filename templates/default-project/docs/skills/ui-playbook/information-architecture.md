# UI Information Architecture

Use this playbook when organizing CodeFlowMu screens, navigation, panels, filters, tabs, or repeated operational workflows.

## Output

```text
# UI Information Architecture

## 1. Primary User Goal
What the screen helps the user do.

## 2. Information Groups

| Group | Purpose | Priority |
|---|---|---|

## 3. Navigation / Flow
How the user moves through the workflow.

## 4. State and Empty States
Loading, empty, blocked, done, error.

## 5. Scan Path
What should be readable first, second, third.

## 6. Risks
Possible confusion, hidden actions, or unsafe shortcuts.
```

## Rules

- Operational tools should be dense but organized.
- Put lifecycle and evidence near the action that depends on them.
- Do not hide destructive or governance actions behind ambiguous controls.
